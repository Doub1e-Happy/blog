用 Next.js 和 MDX 搭建博客是创建快速、开发者友好站点的最佳方式之一。以下是 2026 年的做法。

## 第一步：创建项目

```bash
npx create-next-app@latest my-blog --typescript --tailwind --app --src-dir
```

## 第二步：安装 MDX 依赖

```bash
npm install @next/mdx @mdx-js/loader @mdx-js/react gray-matter
```

## 第三步：配置 `next.config.ts`

```ts
import createMDX from "@next/mdx";

const nextConfig: NextConfig = {
  output: "export",
  pageExtensions: ["js", "jsx", "ts", "tsx", "md", "mdx"],
};

const withMDX = createMDX({ extension: /\.mdx?$/ });
export default withMDX(nextConfig);
```

## 第四步：创建内容目录

```
content/blog/
├── tech/
├── life/
└── tutorials/
```

## 第五步：用 `gray-matter` 加载文章

```ts
import fs from "fs";
import path from "path";
import matter from "gray-matter";

export function getAllPosts() {
  const files = findMdxFiles(path.join(process.cwd(), "content/blog"));
  return files.map(file => {
    const raw = fs.readFileSync(file, "utf-8");
    const { data, content } = matter(raw);
    return { slug: slugFromPath(file), frontmatter: data, content };
  });
}
```

## 部署到 GitHub Pages

添加 GitHub Actions 工作流来构建静态站点并部署到 Pages。构建产物输出到 `out/` 目录——GitHub Pages 直接提供服务。

就这些！现在你有了一个完全静态、极速、免费的博客。再加上 Fuse.js 搜索和 Giscus 评论，一切就绪。
