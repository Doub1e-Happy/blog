泛型是 TypeScript 最强大的特性之一，但也可能让人望而生畏。让我们通过实际示例来理解它。

## 问题所在

考虑一个返回数组第一个元素的函数：

```ts
function first(arr: any[]): any {
  return arr[0];
}

const num = first([1, 2, 3]); // 类型是 `any`——丢失了类型信息！
```

类型丢失了。`num` 是 `any`，这违背了使用 TypeScript 的初衷。

## 泛型的解决方案

```ts
function first<T>(arr: T[]): T | undefined {
  return arr[0];
}

const num = first([1, 2, 3]); // 类型是 `number`
const str = first(["a", "b"]); // 类型是 `string`
```

`<T>` 是一个**类型参数**——它捕获调用者传入的类型，并在整个函数签名中使用。

## 约束泛型

有时需要限制 `T` 的范围：

```ts
function getLength<T extends { length: number }>(item: T): number {
  return item.length;
}

getLength("hello"); // ✅ 5
getLength([1, 2, 3]); // ✅ 3
getLength(123); // ❌ 错误：number 没有 `length`
```

## 要点总结

1. 当类型取决于调用者时，使用泛型
2. 使用 `extends` 约束允许的类型
3. 泛型默认值（`<T = string>`）可以让 API 更友好
4. 不要过度使用——不是每个函数都需要泛型

Happy typing!
